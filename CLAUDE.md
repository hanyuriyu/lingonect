# Lingonect — Claude Code Instructions

## Project

Lingonect is a static web app (hosted on GitHub Pages via `CNAME`) that lets
users create AI-powered flashcard stacks and chat with various translation
engines. The main entry points are plain HTML files in the repo root:

- `flashcards.html` — flashcard stack creation, study UI, spaced repetition
- `chat.html`, `engines.html`, `ratings.html`, `seed-ratings.html` — other
  tools
- `workers/` — Cloudflare Worker source for the AI proxies
  (`claudetranslate`, `openai`, `deepseek`, etc.)

Flashcard data is persisted in Firebase Realtime Database, not in the repo.
Rules live in `firebase-rtdb-rules.json`.

## Flashcard content rule (enforced)

**Every flashcard must have exactly three example sentences on the back. The
third example must be generated by OpenAI, not Claude.**

Concretely, in `flashcards.html`:

- Each card object must carry three populated example fields: `example`,
  `example2`, `example3`.
- `example` is produced by the main Claude translation call.
- `example3` **must** be fetched from the OpenAI worker
  (`OPENAI_WORKER`, e.g. `gpt-4o-mini`) via `generateExampleBatch`.
  Never let Claude fill `example3` directly.
- The card back template (`#example3Section`, `#cardExample3`) must render
  `example3` for every stack — do not gate the third example behind a
  language check such as `isChineseLang(targetLang)`.
- If an example generation call fails, log a warning and leave the field
  empty rather than silently dropping the section; the UI already hides
  empty sections, but the data model still expects the three keys.

This rule applies to:

1. New code that creates flashcards (keep all three example fields wired up).
2. Refactors of `createCardsFromInputs` / `generateExampleBatch` — do not
   remove the OpenAI call or the `example3` assignment.
3. Any migration or backfill script touching existing Firebase records —
   top up to three examples, using OpenAI for the third.

A PostToolUse hook in `.claude/settings.json` validates `flashcards.html`
after every edit and will fail loudly if the third-example wiring is
missing. Fix the code rather than bypassing the hook.

## Conventions

- Plain HTML/CSS/vanilla JS — no build step, no framework. Keep it that way
  unless explicitly asked.
- Firebase config and worker URLs are embedded in the HTML; don't move them
  to a bundler.
- Preserve existing 2-space indentation in HTML/JS.
